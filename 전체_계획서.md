\# 1. Overview (서비스 개요) ## 1.1 서비스 개요 ### 1.1.1 프로젝트명 one-pie (Korean Need A Budget) ### 1.1.2 서비스 정의 one-pie은 **개인 및 가족의 재무 목표를 기반으로 소비 의사결정을 지원하는 재무 관리 서비스**이다. 기존 가계부 앱이 **과거 소비 기록 중심**이라면 one-pie은 다음 질문에 답한다.

```
이 소비를 하면
내 목표가 얼마나 늦어질까?
```

또한 개인 자산 관리뿐 아니라 **가족 또는 부부 단위의 재무 목표 공유 및 관리 기능**을 제공한다. ------ ## 1.2 서비스 핵심 가치 (Value Proposition) one-pie이 사용자에게 제공하는 핵심 가치는 다음과 같다. ### 1.2.1 소비 → 미래 영향 연결 기존 가계부

```
이번달 외식
20만원
```

one-pie

```
외식 소비
→ 목표 영향 분석
```

예

```
노트북 250만원
→ 집 구매 9개월 지연
```

즉 one-pie은 소비를 **장기 목표와 연결**한다. ------ ### 1.2.2 재무 의사결정 지원 기존 금융 앱

```

```


\# 7. 한국 가계부 방식 서비스  # 8. 한국 가계부 vs YNAB 차이 | 구분        | 한국 가계부    | YNAB              | | ----------- | -------------- | ----------------- | | 중심        | 소비 기록      | 예산 관리         | | 데이터 방향 | 과거           | 미래              | | 사용 목적   | 소비 분석      | 소비 통제         | | 질문        | 어디에 돈 썼지 | 이 돈을 써도 될까 | ------ # 9. 사용자 심리 차이 (가계부 vs 예산 기반 관리) 기존 가계부 앱과 예산 기반 관리 서비스는 **사용자가 소비를 바라보는 시점**이 다르다. ------ ## 9.1 일반 가계부에서의 사용자 심리 기존 가계부 앱은 대부분 다음 정보를 제공한다.

```
이번달 외식
→ 28만원
```

사용자가 이 정보를 확인하는 시점은 보통

```
월말
```

이다. 그래서 사용자의 반응은 다음과 같다.

```
아… 이번달 외식 많이 했네
다음달에는 줄여야지
```

하지만 이 시점에서는 이미

```
돈이 모두 지출된 이후
```

이다. 즉 기존 가계부는

```
소비 이후 회고
```

중심 서비스이다. ------ ## 9.2 예산 기반 관리에서의 사용자 심리 (YNAB 방식) 예산 기반 관리에서는 다음 정보를 제공한다.

```
외식 예산
200000

사용
180000

남은 예산
20000
```

이 상황에서 사용자가 카페에 가서

```
아메리카노
5000원
```

을 결제하려고 하면 사용자는 자연스럽게 다음 계산을 하게 된다.

```
외식 예산
20000

커피 구매 후
15000 남음
```

이 순간 사용자는 다음 질문을 하게 된다.

```
오늘 카페를 갈까 말까?
```

즉 예산 기반 UX는

```
소비 이전 의사결정
```

을 유도한다. ------ ## 9.3 핵심 개념: Trade-off (선택) 예산 기반 관리의 핵심 철학은 다음과 같다.

```
돈을 쓰려면
다른 돈을 포기해야 한다
```

예

```

20 → 25
```

를 늘리려면

```

30 → 25
```

```
외식
vs
쇼핑
```

이러한 선택 구조를 YNAB에서는 다음과 같이 표현한다.

```
Roll with the punches
```

즉 예산을 상황에 맞게 조정하면서 **의식적인 소비 선택**을 하게 만든다. ------ ## 9.4 행동 변화 이러한 UX는 실제 사용자 행동 변화를 만든다. 일반 가계부

\# 10. Solution (솔루션) ## 10.1 one-pie의 차별화 전략 one-pie은 다음 세 가지 접근을 결합한다.

```
자동 금융 데이터
+
예산 기반 소비 관리
+
목표 기반 재무 시뮬레이션
```

추가로 **가족 재무 공유 기능**을 제공한다. ------ ## 10.2 서비스 포지셔닝 one-pie은

```
가계부 앱
```

이 아니라

```
가족 재무 의사결정 도구
```

이다. ------ ## 10.3 서비스 구조 개념 (Concept Architecture) one-pie의 데이터 흐름은 다음 구조를 가진다.

```
금융 데이터
(카드 / 계좌)
↓
소비 분류
↓
예산 관리
↓
목표 연결
↓
재무 시뮬레이션
```

이 구조를 통해 one-pie은 다음 기능을 제공한다.

```
소비 기록
→ 예산 관리
→ 목표 영향 분석
→ 재무 의사결정
```

\# 16. UX Strategy (UX 전략) ## 16.1 사용자 행동 유도 UX (Behavior UX) one-pie의 핵심 목표는 단순한 소비 기록이 아니라 **사용자의 소비 의사결정을 변화시키는 것**이다. 기존 가계부 앱은 다음 정보를 제공한다.

```
이번달 외식
20만원
```

하지만 이 정보는 소비가 이미 발생한 이후의 **사후 분석**이다. one-pie은 소비가 발생하는 순간 다음 정보를 제공한다.

```
외식 예산
200000

사용
150000

남은 예산
50000
```

이 정보를 통해 사용자는 소비 전 다음 질문을 하게 된다.

```
외식 예산이 5만원 남았는데
오늘 또 외식할까?
```

이 UX는 사용자가 소비 전에 **한 번 더 생각하게 만드는 효과**가 있다. ------ ## 16.1.1 소비 억제 UX (Budget Pressure) 예산 기반 UX는 사용자에게 **적절한 긴장감(pressure)**을 제공한다. 예시

```
외식 예산
200000

현재 사용
180000

남은 예산
20000
```

사용자가 배달 앱을 열었을 때

```
외식 예산
20000 남음
```

을 확인하게 된다. 이때 사용자는 다음 의사결정을 하게 된다.

```
오늘은 집에서 먹을까?
```

이것이 예산 기반 UX의 핵심이다. ------ ## 16.1.2 목표 지연 UX (Goal Impact) one-pie은 단순한 예산 관리뿐 아니라 **목표 영향 분석 UX**를 제공한다. 예시 사용자 입력

```

250만원
```

시스템 분석

```

2034년

노트북 구매 시

2035년
```

결과 표시

```
이 소비로 인해
집 구매 목표가
약 8개월 지연됩니다
```

이 UX는 사용자가 소비를 **장기 목표와 연결하여 판단**하도록 만든다. ------ ## 16.1.3 가족 재무 영향 UX one-pie은 개인 소비가 **가족 재무 목표에 미치는 영향**을 함께 보여준다. 예시

```
가족 목표
집 구매

목표 금액
5억

현재 가족 자산
1억 1천
```

사용자 소비

```
노트북
250만원
```

결과

```
가족 목표 달성
3개월 지연
```

이 정보는 개인 소비를 **가족 단위 재무 의사결정**과 연결한다. ------ ## 16.1.4 행동 변화 UX 핵심 one-pie이 제공하는 핵심 UX는 다음 세 가지이다.

```
1. 남은 예산 표시
2. 목표 지연 분석
3. 가족 재무 영향 분석
```

이 세 가지 UX를 통해 사용자는 소비 전에 다음 질문을 하게 된다.

```
이 소비가
정말 필요한가?
```
